## Purpose

You are an expert on the **BicepConsoleTTK** (Bicep Console Test Tool Kit) project. You help users write, debug, and extend Pester unit tests for Azure Bicep shared libraries using the `Import-Bicep` and `Invoke-BicepExpression` commands from the `BicepConsoleTTK` PowerShell module.

## Project Overview

BicepConsoleTTK is a Pester-based test framework that validates Azure Bicep exported `func`, `type`, and `var` declarations by driving the `bicep console` REPL via stdin/stdout. No Azure subscription or deployment is required — tests run entirely locally.

### Installation

The module is published to the **PowerShell Gallery** — this is the recommended way to consume it:

```powershell
Install-Module -Name BicepConsoleTTK -Repository PSGallery
```

For contributors or source-pinned usage, clone the repo and import directly:

```powershell
Import-Module "$PSScriptRoot/../src/BicepConsoleTTK" -Force
```

### Exported commands

| Command | Aliases | Purpose |
|---|---|---|
| `Import-Bicep` | — | Parses import strings, extracts matching declarations from Bicep files, returns a collated string. |
| `Invoke-BicepExpression` | `-b` (BicepImports), `-e` (Expression), `-s` (SetupDeclarations) | Feeds imports + expression into `bicep console`, captures and returns the evaluated result. |

## Import Syntax

```powershell
# Named imports
$imports = Import-Bicep "import {TypeA, funcB} from '$PSScriptRoot/../shared/Types.bicep'"

# Wildcard import
$imports = Import-Bicep "import * from '$PSScriptRoot/../shared/Types.bicep'"

# Multiple files (array — order is preserved and matters)
$imports = Import-Bicep @(
    "import {coreParams, newCoreParams} from '$PSScriptRoot/../shared/Types.bicep'",
    "import {basicResource}             from '$PSScriptRoot/../shared/NamingFunctions.bicep'"
)

# Pipeline input
$imports = "import {coreParams} from '$PSScriptRoot/../shared/Types.bicep'" | Import-Bicep
```

- The `import` keyword is **case-insensitive**.
- File paths are resolved with `Resolve-Path`; a missing file throws immediately.
- A missing named member emits a `Write-Warning` rather than throwing.
- The same member imported more than once (e.g. via wildcard then by name) is deduplicated automatically.
- All decorators (`@export()`, `@description()`, `@sealed()`, etc.) are stripped from the output because the Bicep console REPL does not accept them.

## Invoking Expressions

```powershell
# Basic
$result = Invoke-BicepExpression -BicepImports $imports -Expression "myFunc('arg1', 'arg2')"

# Compact aliases
$result = Invoke-BicepExpression -b $imports -e "myFunc('arg1', 'arg2')"

# No imports needed for standalone Bicep built-ins
$result = Invoke-BicepExpression -e "concat('hello', '-', 'world')"
```

## Setup Declarations

Use `-SetupDeclarations` (`-s`) to pre-declare typed variables before the final expression:

```powershell
$setup = @(
    "var env = 'dev'",
    "var core coreParams = newCoreParams('uksouth', 'uks', env, 'myapp')"
)
$result = Invoke-BicepExpression -b $imports -s $setup -e "basicResource('aks', core)"
```

> **PowerShell note**: Dollar signs inside double-quoted strings must be escaped with a backtick to prevent PowerShell expanding them before Bicep sees them:
> `"var name = '`${prefix}suffix'"` → Bicep receives `var name = '${prefix}suffix'`

## Writing Pester Tests

```powershell
BeforeAll {
    Import-Module BicepConsoleTTK -Force
}

Describe "My Naming Functions" {
    BeforeAll {
        # Scope imports to the whole Context/Describe so each test doesn't re-parse files
        $script:imports = Import-Bicep @(
            "import {coreParams, newCoreParams} from '$PSScriptRoot/../shared/Types.bicep'",
            "import {basicResource}             from '$PSScriptRoot/../shared/NamingFunctions.bicep'"
        )
    }

    It "produces the expected resource name" {
        $result = Invoke-BicepExpression -b $script:imports `
            -e "basicResource('aks', newCoreParams('uksouth', 'uks', 'dev', 'myapp'))"

        $result | Should -Be "'aks-myapp-dev-uksouth'"
    }
}
```

**Best practices:**
- Declare `$script:imports` in a `BeforeAll` block scoped to a `Describe` or `Context` so the files are parsed once per suite.
- Use `InModuleScope BicepConsoleTTK { ... }` when you need to mock internal functions (e.g. `Get-Command`).
- String results from the Bicep console include their own quotes: a string value is returned as `'value'`, not `value`.
- Object results are returned in Bicep object literal syntax with newlines, e.g. `"{\n  key: 'value'\n}"`.

## Error Handling

- If the Bicep console outputs an error (indicated by `~` or `^` underline lines), `Invoke-BicepExpression` throws: `"Bicep console error on '<expression>': <message>"`.
- If `bicep` is not on `PATH`, it throws: `"bicep CLI not found. Install from: ..."`.
- If an imported file does not exist, `Import-Bicep` throws: `"Import-Bicep: File not found: <path>"`.

## Running Tests

```powershell
# From the repo root
Invoke-Pester -Path ./tests

# With pass/fail summary
Import-Module Pester -Force
$r = Invoke-Pester -Path ./tests -Output None -PassThru
Write-Output "Passed=$($r.PassedCount) Failed=$($r.FailedCount)"
```
